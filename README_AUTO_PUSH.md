# 깃허브 자동 업로드 가이드 (Auto Push Guide)

이 프로젝트 폴더에는 변경된 코드를 자동으로 깃허브에 업로드할 수 있는 두 가지 편리한 방법이 마련되어 있습니다.

---

## 방법 1. 원클릭 자동 업로드 배치 파일 (`git_auto_push.bat`)
프로젝트 폴더 내에 있는 [git_auto_push.bat](file:///g:/광교지역아동센터_코딩강의/Project_Gomoku/git_auto_push.bat) 파일을 사용하는 방법입니다.

### 💡 사용법
1. 코드를 수정하고 저장합니다.
2. `git_auto_push.bat` 파일을 **더블 클릭**합니다.
3. 콘솔 창이 열리며 아래 작업이 자동으로 수행됩니다.
   - `git add .` (수정된 모든 파일 추가)
   - `git commit -m "Auto-upload: 현재날짜/시간"` (자동 커밋 생성)
   - `git push origin main` (깃허브 업로드)
4. 업로드가 완료되면 3초 후 창이 자동으로 닫힙니다.

> **Tip:** 이 배치 파일의 **바로가기**를 바탕화면에 만들어 두면, 언제든지 바탕화면에서 더블 클릭 한 번으로 업로드할 수 있습니다!

---

## 방법 2. VS Code 확장 프로그램 `GitDoc` 사용하기 (저장 시 자동 업로드)
파일을 수정하고 저장할 때마다 알아서 깃허브로 코드를 올려주는 VS Code 기능을 설정하는 방법입니다.

### 🛠️ 설정 순서
1. **VS Code**를 실행합니다.
2. 왼쪽 메뉴 바에서 **확장(Extensions)** 아이콘을 클릭하거나 단축키 `Ctrl + Shift + X`를 누릅니다.
3. 검색창에 **`GitDoc`**을 검색하고 **Install**을 클릭하여 설치합니다. (퍼블리셔: *Rory O'Kane*)
4. 설치가 완료되면, 하단 파란색 상태 표시줄(Status Bar)에 `GitDoc` 관련 상태(예: `GitDoc: Auto` 등)가 표시됩니다.
5. 이제 파일을 수정하고 저장(`Ctrl + S`)할 때마다 백그라운드에서 자동으로 커밋과 푸시(Push)를 수행합니다.

### ⚙️ 추가 옵션 (자동 저장과 함께 사용하기)
VS Code의 **자동 저장(Auto Save)** 옵션과 `GitDoc`을 함께 사용하면, 타이핑을 멈추는 순간 알아서 저장되고 깃허브로 올라가게 됩니다.
1. `Ctrl + ,`를 눌러 설정(Settings)을 엽니다.
2. `Files: Auto Save`를 검색하여 설정을 `afterDelay`로 변경합니다.
3. 이렇게 하면 코드를 작성하고 몇 초 후에 자동 저장되며, `GitDoc`이 이를 감지하여 자동으로 깃허브에 백그라운드 업로드합니다.
